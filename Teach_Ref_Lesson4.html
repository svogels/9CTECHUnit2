Differentiation Strategies & Teaching Notes

📋 Differentiation & Adjustments for Students with Learning Needs

🧩 For Students with ASD Level 1 (Autism Spectrum Disorder)

Environmental Adjustments:

  • ✓ Provide noise-cancelling headphones during game play
  • ✓ Seat away from high-traffic areas and distracting windows
  • ✓ Use a visual timer for each activity phase (display on board)
  • ✓ Create a quiet corner/space for overwhelm moments

Task Modifications:

  • Pre-filled PMI template with sentence starters:
    Plus: "This game is good because..."
    Minus: "This game could be better if..."
    Interesting: "I noticed that..."
  • Simplified flowchart with pre-drawn shapes; student fills in text only
  • Physical flowchart cards to arrange before drawing
  • Reduce requirements: 2 points per PMI column instead of 3

Support Strategies:

  • 🔄 Check in every 5-7 minutes with specific questions
  • 🎯 Provide a "help card" to raise when stuck
  • 💻 Allow choice of working on paper or digital tools
  • 📝 Provide visual schedule of lesson activities
  • ⭐ Use clear, explicit praise for specific achievements

📖 For Students with Reading Comprehension Difficulties (Dyslexia/Processing Issues)

Visual & Audio Support:

  • 🔊 Use Immersive Reader for all text
  • 📹 Provide video tutorials: How to Make a Flowchart
  • 🎥 Screen record the demonstration for replay access
  • 🎨 Use colour-coding consistently throughout materials

Modified Instructions:

Bullet-point task cards with icons:
🎮 Play game (7 minutes)
➕ Write good things
➖ Write bad things
💡 Write interesting things

Scaffolded Templates:

Game Name: _________
What is the goal? _________
How do you control it? _________
What happens when you win? _________

📝 Teacher Notes & Planning

Pre-Lesson Preparation:

  • ✅ Test all game links on school computers (check firewall restrictions)
  • ✅ Prepare alternative games list for students who finish early: Asteroids, Centipede, Frogger, Donkey Kong
  • ✅ Have paper flowchart templates ready as backup (technology may fail)
  • ✅ Print PMI chart templates for students preferring paper
  • ✅ Set up Jamboard or collaborative space before class

⚠️ Common Challenges & Solutions:

  • Challenge: Students get distracted playing games too long.
    Solution: Set a visible timer, give a 2-minute warning, and clearly state the analysis task before they begin.
  • Challenge: Flowcharts become too complex or messy.
    Solution: Limit to 8-10 steps maximum initially. Emphasise starting with the main loop.
  • Challenge: Students struggle with PMI distinctions.
    Solution: Do a class example together first using a very simple game like Pong.
  • Challenge: Some game sites are blocked.
    Solution: Have offline versions, alternative links, or gameplay videos ready as a backup.

🔑 Space Invaders Analysis - Teacher Answer Key

Comprehensive Game Loop Processes

This list covers the core processes that happen repeatedly inside the main game loop of a classic Space Invaders game. This list is for teacher reference to guide discussion.

  • Check Player Input: Has the left/right arrow key or the fire button been pressed?
  • Update Player State: Move the player's ship if a key is pressed, checking for screen boundaries. Create a bullet if fire is pressed.
  • Update Alien Swarm State: Move the entire block of aliens. If any alien hits a screen edge, reverse the swarm's direction and move it down.
  • Update Projectile State: Move player and alien bullets. Remove any that go off-screen.
  • Collision Detection: Check if the player's bullet hits an alien, or if an alien bullet hits the player or a shield.
  • Handle Consequences: If a collision occurs, remove objects (alien, bullet), update the score, decrease lives, or damage shields.
  • Alien Firing Logic: On a timer or by random chance, have a random alien fire a bullet.
  • Check Win/Loss Conditions: Have all aliens been cleared? Have aliens reached the bottom? Is the player out of lives?
  • Render Everything: Clear the old frame and draw all game objects in their new positions.

🕹️ Arcade Game Loop Guides

👻 Pac-Man

  • Check Player Input: Check if an arrow key has been pressed. Pac-Man's movement is continuous, so the game only needs to know the *last* valid direction pressed.
  • Update Pac-Man State: Move Pac-Man one step in his current direction. Check if the new position is a wall; if so, stop movement.
  • Update Ghost State: Each of the four ghosts updates its position based on its unique AI (e.g., Blinky chases directly, Pinky tries to get in front). Their state (Chase, Scatter, Frightened) determines their target.
  • Collision Detection & Consequences:
    • Pac-Man vs. Pellet/Power Pellet: If Pac-Man is on a tile with a pellet, remove the pellet and increase the score. If it's a Power Pellet, also change the ghosts' state to 'Frightened'.
    • Pac-Man vs. Ghost: Check if Pac-Man's tile is the same as any ghost's tile.
      • If the ghost is in Chase/Scatter mode, the player loses a life.
      • If the ghost is in 'Frightened' mode, the ghost is "eaten" (sent back to the central pen) and the score increases.
  • Check Win/Loss Conditions: Have all pellets been eaten? (Level complete). Have the player's lives reached zero? (Game over).
  • Render Everything: Draw the maze, remaining pellets, Pac-Man, and the ghosts in their new positions. Update the score and lives display.

🐍 Snake

  • Check Player Input: Check for arrow key presses. The new direction cannot be the direct opposite of the current direction (e.g., can't go left if currently moving right).
  • Update Snake State:
    • Determine the new position for the snake's head based on the current direction.
    • Insert the new head position at the front of the snake's body list/array.
    • Remove the last segment of the snake's tail. (This is skipped if the snake just ate food).
  • Collision Detection & Consequences:
    • Snake Head vs. Food: If the head's new position is the same as the food's position, increase the score, and generate a new piece of food at a random, empty location. The snake's tail is NOT removed this frame, making the snake longer.
    • Snake Head vs. Walls: If the head's new position is outside the screen boundaries, the game is over.
    • Snake Head vs. Snake Body: Check if the head's new position is the same as any of its own body segments. If so, the game is over.
  • Render Everything: Clear the screen and draw all segments of the snake's body and the piece of food. Update the score display.

✈️ Jet Fighter (1975 2-Player version)

  • Check Player 1 Input: Check for rotation, thrust, and fire button presses for Player 1.
  • Check Player 2 Input: Check for rotation, thrust, and fire button presses for Player 2.
  • Update Player 1 State: Apply physics. If thrusting, increase velocity in the jet's current direction. Update the jet's position based on its current velocity. If firing, create a bullet.
  • Update Player 2 State: Apply the same physics and updates for Player 2.
  • Update Projectile State: Move any active bullets across the screen based on their initial direction and speed.
  • Collision Detection & Consequences:
    • Player 1 Bullet vs. Player 2 Jet: If a collision occurs, increase Player 1's score and reset Player 2's jet to its starting position.
    • Player 2 Bullet vs. Player 1 Jet: If a collision occurs, increase Player 2's score and reset Player 1's jet to its starting position.
  • Check Game Timer: Decrease the round timer by one tick. If the timer reaches zero, end the round and display the final scores.
  • Render Everything: Draw both jets, any bullets, and the score for both players, as well as the remaining time.
