10 minsUnderstanding Pygame Coordinates

Interactive Demonstration: The Coordinate System

In Pygame (and most computer graphics):

  • (0, 0) is at the TOP-LEFT corner
  • X increases → as you move RIGHT
  • Y increases ↓ as you move DOWN

Mouse Position: (0, 0)

Click to place a rectangle at that position!

Key Concept: Think of the screen as graph paper turned upside down!

20 minsI Do: Teacher Demonstration

Live Coding Demo: Drawing and Moving a Rectangle

import pygame
import sys

# Initialize Pygame
pygame.init()

# Constants
WINDOW_WIDTH = 800
WINDOW_HEIGHT = 600
FPS = 60

# Colors (R, G, B)
BLACK = (0, 0, 0)
GREEN = (0, 255, 0)

# Set up the display
screen = pygame.display.set_mode((WINDOW_WIDTH, WINDOW_HEIGHT))
pygame.display.set_caption("Drawing and Movement Demo")

# Player variables
player_x = 400  # Start in middle of screen
player_y = 500  # Near bottom like Space Invaders
player_width = 50
player_height = 30
player_speed = 5  # How fast the player moves

# Game variables
clock = pygame.time.Clock()
running = True

I Do: Main Game Loop

# Game loop
while running:
    # 1. HANDLE EVENTS (INPUT)
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False
    
    # 2. HANDLE CONTINUOUS KEYBOARD INPUT
    keys = pygame.key.get_pressed()
    
    # Move left
    if keys[pygame.K_LEFT]:
        player_x -= player_speed
        # Boundary check - don't go off left edge
        if player_x < 0:
            player_x = 0
    
    # Move right
    if keys[pygame.K_RIGHT]:
        player_x += player_speed
        # Boundary check - don't go off right edge
        if player_x > WINDOW_WIDTH - player_width:
            player_x = WINDOW_WIDTH - player_width
    
    # 3. DRAW EVERYTHING (OUTPUT)
    screen.fill(BLACK)
    pygame.draw.rect(screen, GREEN, (player_x, player_y, player_width, player_height))
    
    # Update display
    pygame.display.flip()
    clock.tick(FPS)

pygame.quit()
sys.exit()

Key Teaching Points:

  • Drawing shapes: pygame.draw.rect(surface, color, (x, y, width, height))
  • Keyboard methods: event.KEYDOWN vs get_pressed()
  • Boundary detection: Keep player on screen!

5 minsExit Ticket & Reflection

Quick Assessment

1. Code Understanding

What is the difference between these two methods?

# Method A
if event.type == pygame.KEYDOWN:
    if event.key == pygame.K_SPACE:
        shoot()

# Method B  
keys = pygame.key.get_pressed()
if keys[pygame.K_SPACE]:
    shoot()

2. Coordinate System

If a rectangle is at position (100, 200) and we add 50 to its X coordinate, which direction does it move?

Teacher Notes & Differentiation

Preparation Checklist:

  • Ensure Grok Learning accounts are active
  • Have backup movement code ready

Common Issues & Solutions:

  • Import errors: Check pygame spelling, restart IDE
  • Black screen: Ensure fill() and flip() are called
  • No movement: Verify event loop indentation

Differentiation Strategies

For Students Needing Support:

  • Provide partially completed code with comments
  • Focus on horizontal movement only
  • Use pair programming

For Advanced Students:

  • Implement smooth acceleration
  • Add rotation to player shape
  • Research sprite classes
