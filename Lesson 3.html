10 mins3. Understanding Pygame Coordinates

Interactive Demonstration: The Coordinate System

In Pygame (and most computer graphics):

  • (0, 0) is at the TOP-LEFT corner
  • X increases → as you move RIGHT
  • Y increases ↓ as you move DOWN

Mouse Position: (0, 0)

Click to place a rectangle at that position!

Key Concept: Think of the screen as graph paper turned upside down!

20 mins4. I Do: Teacher Demonstration

Live Coding Demo: Drawing and Moving a Rectangle

Teacher demonstrates while explaining each line:

import pygame
import sys

# Initialize Pygame
pygame.init()

# Constants
WINDOW_WIDTH = 800
WINDOW_HEIGHT = 600
FPS = 60

# Colors (R, G, B)
BLACK = (0, 0, 0)
WHITE = (255, 255, 255)
RED = (255, 0, 0)
GREEN = (0, 255, 0)
BLUE = (0, 0, 255)

# Set up the display
screen = pygame.display.set_mode((WINDOW_WIDTH, WINDOW_HEIGHT))
pygame.display.set_caption("Drawing and Movement Demo")

# Player variables
player_x = 400  # Start in middle of screen
player_y = 500  # Near bottom like Space Invaders
player_width = 50
player_height = 30
player_speed = 5  # How fast the player moves

# Game variables
clock = pygame.time.Clock()
running = True

# Game loop
while running:
    # 1. HANDLE EVENTS (INPUT)
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False
        elif event.type == pygame.KEYDOWN:
            if event.key == pygame.K_ESCAPE:
                running = False
    
    # 2. HANDLE CONTINUOUS KEYBOARD INPUT
    keys = pygame.key.get_pressed()
    
    # Move left
    if keys[pygame.K_LEFT] or keys[pygame.K_a]:
        player_x -= player_speed
        # Boundary check - don't go off left edge
        if player_x < 0:
            player_x = 0
    
    # Move right
    if keys[pygame.K_RIGHT] or keys[pygame.K_d]:
        player_x += player_speed
        # Boundary check - don't go off right edge
        if player_x > WINDOW_WIDTH - player_width:
            player_x = WINDOW_WIDTH - player_width
    
    # 3. DRAW EVERYTHING (OUTPUT)
    # Clear screen
    screen.fill(BLACK)
    
    # Draw the player rectangle
    pygame.draw.rect(screen, GREEN, (player_x, player_y, player_width, player_height))
    
    # Draw some reference points to understand coordinates
    # Top-left corner (0,0)
    pygame.draw.circle(screen, RED, (0, 0), 5)
    
    # Center of screen
    center_x = WINDOW_WIDTH // 2
    center_y = WINDOW_HEIGHT // 2
    pygame.draw.circle(screen, WHITE, (center_x, center_y), 3)
    
    # Update display
    pygame.display.flip()
    clock.tick(FPS)

# Clean up
pygame.quit()
sys.exit()

Key Teaching Points:

  1. Drawing shapes: pygame.draw.rect(surface, color, (x, y, width, height))
  2. Keyboard input methods:
    • event.type == pygame.KEYDOWN - Detects when key is first pressed
    • pygame.key.get_pressed() - Checks if key is currently held down
  3. Boundary detection: Using if statements to keep player on screen
  4. Coordinate system: Remember (0,0) is top-left!

Common Mistakes to Address:

  • Forgetting to clear the screen (creates trail effect)
  • Not updating display with flip()
  • Mixing up x/y coordinates

5 mins9. Exit Ticket & Reflection

Exit Ticket: Drawing & Movement

1. Code Understanding

What is the difference between these two methods of detecting keyboard input?

# Method A
if event.type == pygame.KEYDOWN:
    if event.key == pygame.K_SPACE:
        shoot()

# Method B  
keys = pygame.key.get_pressed()
if keys[pygame.K_SPACE]:
    shoot()
  • [ ] Method A detects single press, Method B detects held keys
  • [ ] They do exactly the same thing
  • [ ] Method A is for movement, Method B is for actions
  • [ ] Method B is faster than Method A

2. Coordinate System

If a rectangle is at position (100, 200) and we add 50 to its X coordinate, which direction does it move?
