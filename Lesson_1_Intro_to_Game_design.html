Your Assessment Task: Game Design Portfolio

Your Mission: Design a simple, single-screen game

Portfolio Must Include:

  • Design Brief
  • Game Loop Diagram
  • Visual Mockups
  • User Testing Plan
  • Reflection

Your Turn: Start Your Design Brief!

Section 1: Game Concept (10 mins)

  • Game name
  • One paragraph description
  • Genre selection (e.g., Arcade, Puzzle, Action)

Section 2: Target Audience (10 mins)

  • Age group
  • Gaming experience (e.g., Casual, Expert)
  • Player interests

Section 3: Design Constraints (15 mins)

  • Technical limits
  • Time limits
  • Skill limits
  • Content appropriateness

Design Brief Example

Name: "Asteroid Dodger"

Description: A survival game where players control a spaceship navigating through an asteroid field. The longer you survive, the higher your score. Speed increases over time!

Genre: Action/Arcade

Target Audience:

  • Ages: 10-16
  • Experience: Casual gamers
  • Interests: Space, quick challenges

Every Game Has Limits!

Technical Constraints: 2D graphics only, single screen (no scrolling), keyboard input only.

Time Constraints: 6 weeks to design, must be playable in under 2 minutes.

Skill Constraints: Your current programming knowledge, available tools (e.g., Scratch, GDevelop).

Why Constraints Matter: They force creativity!
