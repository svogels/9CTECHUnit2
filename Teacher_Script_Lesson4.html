Part 2 Demonstration (10 minutes)

Introduction to Flowchart Shapes

"Right everyone, now we're going to learn about flowcharts - these are like maps that show how our game thinks and makes decisions. Before we start drawing, let me show you the special shapes we use in flowcharts. Each shape has a specific meaning - it's like a secret code that all programmers understand."
[Draw on board or display on screen]

1. Start/End Points - OVAL

STARTEND
"First, we have the OVAL. This is always used for START and END points. Think of it like the power button on your game console - it turns things on and off. Every flowchart begins with START in an oval and finishes with END in an oval."

2. Process/Action - RECTANGLE

Move Player Left
"Next, we have the RECTANGLE. This is our action box. Whenever the computer needs to DO something, we put it in a rectangle. For example: 'Move player left', 'Add 10 points to score', or 'Display game over screen'."

3. Decision - DIAMOND

Spacebar?
"Now, here's the most important one for games - the DIAMOND. This is our decision maker. Whenever the game needs to ask a question that has a YES or NO answer, we use a diamond. Questions like 'Did the player press spacebar?', 'Did the bullet hit an enemy?', or 'Is the score greater than 100?' The diamond always has two paths coming out of it - one for YES and one for NO."

4. Input/Output - PARALLELOGRAM

Get Keyboard Input
"We also have the PARALLELOGRAM. This represents INPUT or OUTPUT. When the player types something, that's input. When the game shows something on screen, that's output. For our simple games, we often just use rectangles for these actions too, and that's okay."

5. Flow Arrows

"Finally, we connect everything with ARROWS. These show the flow - the order things happen in. Think of arrows as the roads that connect different parts of our game's logic."

Space Invaders Flowchart Example

"Let me show you how this works with a simple version of Space Invaders:"
[Begin drawing flowchart on board/screen]

Step-by-Step Construction:

"1. We start with an OVAL: 'START'
2. Then a RECTANGLE for our first action: 'Load game graphics'
3. Another RECTANGLE: 'Create player ship at bottom'
4. Another RECTANGLE: 'Create enemy grid at top'
5. Now we enter our game loop. It starts with a DIAMOND - a decision: 'Is Game Over?'"
[Draw arrow labeled "NO" to continue the loop, and "YES" to the end]
" - If NO, the game continues. We have a RECTANGLE: 'Check for player input'
6. Another DIAMOND: 'Did player press left arrow?'
- If YES (draw arrow): RECTANGLE 'Move ship left'
7. Another DIAMOND: 'Did player press spacebar?'
- If YES: RECTANGLE 'Fire bullet'
8. RECTANGLE: 'Move all enemies'
9. RECTANGLE: 'Move all bullets'
10. DIAMOND: 'Did bullet hit enemy?'
- If YES: RECTANGLE 'Remove enemy' then 'Add to score'
11. DIAMOND: 'Did enemy reach player?'
- If YES: RECTANGLE 'Set Game Over to TRUE'
12. Finally, an arrow loops from the last step back up to the 'Is Game Over?' diamond. This is the GAME LOOP.
- If 'Is Game Over?' is YES: RECTANGLE 'Show Final Score' then OVAL 'END'"

Key Teaching Points

"See how the shapes tell us exactly what type of instruction each step is? The diamonds are where the game makes choices, and those choices change what happens next. That's what makes games interactive!"

Reinforcement Questions:

  • "What shape would we use for 'Calculate final score'?" (Rectangle - it's an action)
  • "What shape for 'Is health zero?'" (Diamond - it's a yes/no question)
  • "What shape for 'Game Over'?" (Could be a rectangle for displaying the screen, or an oval for the final END point)

Common Student Mistakes to Address:

  • Using rectangles for decisions - remind them diamonds are for questions.
  • Forgetting to label arrows from diamonds with YES/NO.
  • Not including an END point or not having a loop.
  • Making the flowchart too complex initially - start simple!
